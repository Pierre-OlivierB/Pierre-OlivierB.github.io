Utilisation de GitHub

GitHub en MULTI

Sur GitHub

Après avoir créé un repository :

Click droit :GitBashHere sur le poste de travail par exemple.

"uploading an existing file" dans l'onglet bleu de Quick setup en bas.

Drag and drop avec du travail déjà fait. Ex: dossier img, html, css.

Ce sera le dossier de la personne qu'on pourrait appeller MAIN.

Les autres vont gravités autour.

Pour ça tout le monde va devoir récupérer le lien du fichier.

Sur notre PC:

1-Git Bash here sur le poste de travail

2- $git clone lienDuTravailSurGithub
\\Création du dossier sur le poste de travail.

3-GitBashHere sur le dossier nouvellement créé.

4- code .
\\permet d'ouvrir VSC en prenant en compte les composants.

Il faut maintenant donné des autorisations

Sur GitHub:

5- Sur le projet en question allez sur :Settings (à droite dans la barre où est <>Code, Issues,...)

6- Manage access

7- Invite a collaborator
\\Entrez le pseudo de la pesonne voulu (ça peut être envoyé par mail)

La personne doit accepter
\\En attente

Autorisation faite, les personnes liées au projet peuvent désormais push!!!

Sur notre PC:

9- $git add .

10- $git commit -m "message à mettre pour savoir à quoi ça correspond"

11- $git push -u origin main

12- Les changments ont été fait sur Github

Faire un readme
pour aider à comprendre le projet.

13- Ajouter les infos importantes

14- # = h1; ##=h2; ``=grise le mot entre apostrophe

Faire un fichier .gitignore

15- Ajouter les fichiers qui ne sont pas pris en compte par le commit

16- Créez un fichier .gitignore à partir de VsCode par exemple.

17- Liste les fichier à ne pas prendre en compte
\\ex: *.txt ; l'idée est de ne pas commit des fichiers volumineux.
