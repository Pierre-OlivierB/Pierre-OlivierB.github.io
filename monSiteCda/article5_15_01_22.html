Utilisation de GitHub

GitHub en SOLO

Marche à suivre

Après avoir créée son dossier sur git hub. Nous prenons(copier) le lien donné dans la quick installation.

Click droit :GitBashHere sur le poste de travail par exemple.

Dans l'onglet qui s'ouvre :

1- $ git clone https://github.com/...
\\ cela créé un dossier vide qui provient git hub, sauf si on y a déjà mis quelque chose!!! Si on a un travail déjà fait qu'on voudrait mettre dans ce dossier alors créée:

2- Glissé déposé dans le nouveau dossier.
Attention nous somme toujours sur le poste de travail! Il nous faut donc aller sur:

3-$ cd nomDuFichierGithub/
\\ on est bien sur la branche master

4-$ git add .
\\ on ajoute tout ce qui a dans le dossier (merci à la black liste, si non tout est ajouté)

5-$ git status

Maintenant on envoit notre travail sur le serveur (GitHub)

6-$ git commit -m "Premier envoi"
\\le commit ne travail qu'en local il nous faut donc une autre manip en plus!

7-$ git push -u origin main
\\ça nous ouvre un onglet (sauf si on a enregistrer notre mdp sur GitHub).

Pourquoi cette démarche?
Nous essayons d'envoyer des fichiers sur un serveur, la protection veut qu'il sache qui nous sommes. Avant d'ouvrir la porte nous demandons :"Qui c'est?", après avoir rentré notre code c'est comme si on lui répondait :"C'est moi!" la réponse qui à toujours du sens.

Pendant cet apparté nous voyons des écritures s'afficher sur l'onglet de commands. Il affiche plein de détails et finit par nous redonner la possibilité d'écrire des commandes.

Allons sur GitHub pour voir ce qui s'est passé:

Après un F5 de la page <>Code, on se retrouve avec nos fichiers push (précision nous sommes sur la branche master, affiché en haut à gauche, en dessous de l'onglet <>Code).

A gauche le nom du fichier

Au centre le commentaire du push

A droite le timecode du commit effectué

D'autres informations s'affichent sur la droite About, Releases, Packages, Languages qui peuvent avoir leur importances. Tout dépend de ce qu'on cherche.
