Utilisation de Git

Git

C'est quoi?

Git, c'est du "versionning" d'un travail ciblé. Ce sont des copies faites avec un code donné, pour ciblé une version à cause d'un problème rencontré par exemple.

Pour la suite allez télécharger git sur -->GIT<-- !

Pas à pas pour utiliser git:

1- clic droit dans le dossier cible :
Git bash here.

Ligne de commande:

2- git config –global user.name "Name"

3- git config –global –user.email "email.mail@mail.mel"

4- git init

5- git status
//: pour connaître le statu du travail

6- git add
//: ajoute un fichier à suivre (attention à "git add ." qui référence tout)

7- git status
//: pour vérifier les ajouts

8- git commit
//: commit le travail surveillé (sauvegarde local)

une page s'ouvre

{'git commit -m "ici c'est le commentaire" ' permet de mettre le commentaire}

9- ajouter :titre du commit + escp pour partir + ":wq" //: ":wq" permet de sauvegarder

10- git status
//: tout doit disparaître.

Conflits!!

Deuxième possibilité: Branche

A- git branch v2.0

A1- git status
//: on est toujours sur la branche master

A2- git checkout v2.0
//: s'aligner sur la branche ici v2.0

A3- git checkout v2.0 -b v2.1
//raccourci qui permet de créer un new branch et de s'y aligner

A4- git checkout master

si il y a eu des commit ou du travail fait sur cette nouvelle version et qu'on vont push cette branch sur git hub:

A5- git push origin v2.0
//: c'est une push de la branche v2.0 sur la branch origin => une nouvelle branche se créé sur le distant (voir git hub)

A6- apparition d'un page pour le code si vous n'avez pas enregistrer votre code.

Fusion branche:

en position master:

AA1- git merge 2.0
//: la branche master a les modifs du 2.0 (les deux entités sont quand même séparé)

AA2- git tag -a 2.0 -m "Mise en production blablabla" master
//:ici un tage est mis sur la branche master pour indiquer le merge de la v2.0 sur la version master.

!!!!ne pas utiliser le même nom que la branche si non confusion pour git

AA3- git push origin master --tags

Il faut penser à push les tags également , push tags seul ne marche pas.

Ajouté à l'onglet release sur git hub

Pour aller plus loin : Travail en équipe!

1- git log
\\ montre les commits effectués

2- git log -n2
\\ montre les deux derniers commit

3- git show sha-1
\\le sha-1 (dit chat one) donne le code de la version du commit, il est long

4- git checkout sha-1
\\selectionne la version désiré et change la version actuel du travail. Attention nou ne serons alors plus sur la branche Master! Pour y retourner =>

5- git checkout master
\\ c'est la version head, la dernière version produite!
