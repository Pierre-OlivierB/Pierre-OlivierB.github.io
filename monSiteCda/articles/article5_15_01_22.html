Utilisation de GitHub en MULTI

Travailler sur la branche

Marche à suivre

Toujours penser aux vérifications avant de faire quoi que ce soit!

Récap.: add . + commit -m "" +

1- $git push -u origin nomDeLaBranche

Fusionner :

Sur notre PC:

2- $git merge nomDeLaBranche (fusion de la branche avec le main)

Sur GitHub: (recommandé)

2 (bis)- Sur le dossier du travail en dessous de la barre de navigation où se trouve <>Code on trouve le décompte des branches "1 branch" avec un petit arbre devant. On clique dessus.

3- Cela nous dirige vers une page où toutes nos branches son affichées. Sur chacune, à part sur le main évidemment, apparaît un bouton : "New pull request"

4- On arrive sur une page ou les différences des fichiers apparaissent. Rouge (avant), Vert (nouveau);

5- On peut écrire un commentaire.

6- Après vérifications:
Create pull request (bouton vert en bas à droite des commentaires écrits)

Si conflits:

7- Le déroulé de la création s'affiche, et un conflit également!

8- On clique sur
Resolve conflicts à droite dans l'onglet de l'erreur.

9- Une nouvelle page s'ouvre, dévoilant notre code en texte, avec les erreurs encadrées de Jaune reliées d'une ligne Rouge

10- On supprime à la main sur cette interface ce qui ne nous convient pas.

11- En haut à droite,
APRES
avoir résolu le.s conflit.s, on clique sur :

Mark as resolved

12- Une encoche verte apparaît alors sur la droite de notre fichier.

13- D'aurtes arbitrages à FAIRE peuvent apparaître sur la gauche de la pages.

14- Après avoir tout résolu,
en haut à droite sur un bouton vert :

Commit merge

15- Cette fois les conflits sont résoluts, sur notre historique.
La validation est faite.
Il ne nous reste qu'à appuyer sur le bouton vert en bas à gauche de l'onglet où est écrit
"This branch has no conflicts with the base branch":

Merge pull request

16- Confirm merge (bouton vert)!

17- Pull request successfully merged and closed. (en violet)
